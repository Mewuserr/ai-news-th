## ฟีเจอร์ที่ทำเสร็จแล้ว (ครบตามที่คุยกันแล้วทั้งหมด)
- หน้าเว็บ: วันนี้ (index), รายสัปดาห์, คลังข่าวย้อนหลัง (มีค้นหา), สถิติ (word cloud, leaderboard, region breakdown, week-over-week), ไทม์ไลน์ข่าวใหญ่, ควิซ, สรุปประจำเดือนแบบ Wrapped (คลิกได้, มีรายละเอียด), บันทึกไว้อ่าน
- ธีม: พื้นดำสนิท + ดาวกระพริบเคลื่อนไหว (canvas) + ดาวเคราะห์จริงจาก NASA ลอยรอบจอ (โลก, ดวงจันทร์, ดวงอาทิตย์, ดาวอังคาร, ดาวเสาร์มีวงแหวน, ดาวยูเรนัส — รูปถ่ายจริง public domain ทั้งหมด ไม่ใช่ vector) + สี ฟ้า(#4fd1ff)/ชมพู(#ff4fd8) + ฟอนต์ Leelawadee UI (ไทย-อังกฤษสมดุล) + warp-speed transition ตอนเปลี่ยนหน้า
- **🚀 ยานอวกาศดวลกันแบบ Canvas บนหน้าแรก** (แทนที่การ์ดวงกลมเดิมและแทนที่ battle-intro แบบเก่าที่หายไปหลังโหลด) — วาดด้วย `<canvas>` จริง มีลำตัว ปีก กระจกห้องนักบิน เปลวไฟท้ายยาน บินหลบเคลื่อนไหวตลอดเวลา ไม่หยุด:
  - ขนาดยาน + ความถี่การยิง สะท้อน **"พลัง" จริง** (สูตร: ข่าวใหญ่×5 + ข่าวรวม) — อันดับ 1 ใหญ่สุด ยิงถี่สุด
  - ยานรองๆ ยิงเล็งไปที่ตัวที่แรงกว่าเป็นหลัก (มีเหตุผล ไม่สุ่มมั่ว)
  - มีป้ายชื่อ+เหรียญ+ตัวเลขพลังลอยกำกับข้างยานแต่ละลำในฉากเลย
  - ยังมีการ์ดข้อมูลเดิม (ชื่อ/คลาส/พลัง/รายละเอียด) อยู่ด้านล่างครบ
  - โค้ดอยู่ที่ `DOGFIGHT_SCRIPT` + `dogfight_html()` ใน `scripts/generate.py`
  - **เสร็จแล้ว ผู้ใช้อนุมัติแล้ว ("ผมโอเคแล้วคุณ") ไม่ต้องคุยต่อ**
- ฟีเจอร์ผู้ใช้: บุ๊กมาร์ก, ไลก์/ไม่ชอบ, ทำเครื่องหมายอ่านแล้ว, แชร์, แปลอังกฤษ, ฟังทีละข่าว/ฟังบันทึกไว้ทั้งหมด/ฟัง 2 พิธีกร (ในเว็บ ใช้ Web Speech API), ติดตามเฉพาะบริษัท, สุ่มข่าวเก่า, อ่านเร็ว 60 วิ, โหมดเล่าเป็นเรื่องราว (รอ AI เขียนรอบข่าวจริงถัดไปถึงจะมีข้อมูล), Export ปฏิทิน .ics, QR โค้ด, เลือกสีธีม, ระบบยศ/สตรีค

## บั๊กที่แก้ไปแล้ว (2026-07-13)
- **ข่าวไม่อัปเดตเลยตั้งแต่สร้างเว็บ**: cloud routine หลักยิงตรงเวลาทุกวันจริง (ยืนยันจาก `last_fired_at` ผ่าน RemoteTrigger) แต่ไม่มีข่าวใหม่ถูกเขียน/push เลยตั้งแต่ 2026-07-11 — จุดอ่อนที่ชัดที่สุดคือ engine-prompt.md เดิมสั่งให้ "ข้ามทุกอย่างเงียบๆ" ถ้า agent สรุปว่าไม่มีข่าวใหม่ ไม่มีร่องรอยอะไรให้เช็คได้เลยว่ารันจริงหรือไม่ แก้โดย (1) เพิ่มขั้นตอน 9 บังคับให้เขียน+commit+push `data/engine-log.md` ทุกรอบไม่ว่าผลจะเป็นยังไง (2) ปรับ prompt ให้ agent ไม่ด่วนสรุปว่าไม่มีข่าวง่ายเกินไป (3) ตั้ง routine สำรอง `ai-news-th-daily-engine-backup` รันตี 6 คอยเช็คและรันซ้ำแทนถ้าตัวหลักพลาด — ทดสอบรันมือทั้ง pipeline (ค้นข่าว→เขียน JSON→generate.py→push) สำเร็จไม่มี error เลย ยืนยันว่า mechanics ไม่ได้พัง ปัญหาอยู่ที่ตรรกะของ agent ตอนตัดสินใจว่ามีข่าวไหม
- ถ้าผู้ใช้ถามอีกว่าทำไมข่าวไม่อัปเดต **เช็ค `data/engine-log.md` ก่อนเป็นอันดับแรก** จะเห็นทันทีว่า routine ล่าสุดรันวันไหน สำเร็จไหม ไม่ต้องเดา
- **เพิ่มหมวดข่าว "อวกาศ"** ตามคำขอผู้ใช้ — ครอบคลุมทั่วโลก (NASA, ESA, CNSA, JAXA, ISRO, SpaceX, Blue Origin, Rocket Lab) ไม่ใช่แค่ตะวันตก และเพิ่มแหล่งข่าว AI จีน/เอเชียที่ขาดไปด้วย (DeepSeek, Qwen, Zhipu, Moonshot) พร้อมเดินสายไฟหมวดใหม่เข้ากับ champion class (`CATEGORY_CLASS`) และ region stats (`COMPANY_REGION`) ที่ตกหล่นไปตอนแรก
- **ตรวจทั้งเว็บหาบั๊กแฝง** (คำขอผู้ใช้) เจอ 2 จุดจริง: (1) `stars_html()` ใช้ `round()` ของ Python ที่ปัดแบบ banker's rounding ทำให้ดาว 3.5 กับ 4.5 แสดงผลจำนวนดาวเท่ากัน (เจอชนจริงใน capabilities.json: Lyria 3.5 vs Udio 4.5) แก้เป็น `int(x+0.5)` (2) `context_th` ถ้า AI เขียนเป็น `null` ตรงๆ (ไม่ใช่แค่ไม่ใส่ฟิลด์) จะ crash ตอน generate.py เพราะ `.get(key, default)` เช็คแค่ key หาย ไม่เช็คค่า None — ยังไม่เคยเกิดจริงแต่เป็นไปได้สูงเพราะ AI เขียน JSON เอง แก้ป้องกันไว้แล้ว
