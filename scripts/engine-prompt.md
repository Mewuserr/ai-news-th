# AI News Thai Engine — daily routine prompt

รันทุกวันเวลา 08:00 (Asia/Bangkok) โดย **cloud scheduled routine** (Claude Code cloud agent — รันบน sandbox ของ Anthropic ไม่ใช่บนเครื่อง PC ของผู้ใช้) routine นี้ clone repo `https://github.com/Mewuserr/ai-news-th` มาทำงานที่ root ของ repo โดยตรง (ไม่ใช่ path `E:\ai-news-th` ซึ่งเป็น path บนเครื่อง PC เท่านั้น — repo กับโฟลเดอร์บนเครื่องนี้ sync กันผ่าน git)

**สำคัญ: routine นี้รันบน cloud sandbox ไม่มีหน้าจอ Windows ให้เชื่อมต่อ — ข้าม "ขั้นตอน 7 แจ้งเตือน" ไปเลย ไม่ต้องพยายามรัน `python scripts/notify.py`** การแจ้งเตือนบนเครื่อง PC จริงทำโดย Windows Task Scheduler แยกต่างหาก (ดู `scripts/notify_check.py` ที่รันในเครื่อง ไม่เกี่ยวกับ routine นี้)

ทำตามขั้นตอนนี้ทั้งหมดในรอบเดียว:

## 0. เตรียมตัว
- อยู่ที่ root ของ repo ที่ clone มาแล้ว (จาก git checkout ของ routine)
- หาวันที่ปัจจุบันตามเวลาไทย (Asia/Bangkok, UTC+7) รูปแบบ `YYYY-MM-DD` — ใช้ค่านี้เป็นชื่อไฟล์ข่าววันนี้ตลอดทั้งรอบ เรียกว่า `{today}`
- อ่าน `data/latest.json` (ถ้ามี) เพื่อรู้ว่าข่าวไหนเคยเก็บไปแล้ว (เทียบด้วย `url`) กันไม่ให้ซ้ำ

## 1. ค้นข่าว AI ล่าสุด
ค้นหาข่าวที่ประกาศ/เผยแพร่ภายใน ~24-48 ชั่วโมงที่ผ่านมา จากสองกลุ่มแหล่งข่าว:

**แหล่งทางการ (เช็คตรงหน้าข่าว/บล็อกของแต่ละเจ้า ผ่าน WebFetch หรือ WebSearch `site:` ก็ได้):**
- OpenAI — openai.com/news
- Anthropic — anthropic.com/news
- Google DeepMind — deepmind.google/discover/blog
- Meta AI — ai.meta.com/blog
- Microsoft AI — blogs.microsoft.com/ai, news.microsoft.com
- xAI — x.ai/news
- Mistral AI — mistral.ai/news
- Hugging Face — huggingface.co/blog

**สื่อเทคทั่วไป (ใช้ WebSearch หา "AI news today" หรือ site-specific):**
- TechCrunch (AI section)
- The Verge (AI section)
- VentureBeat (AI section)
- Ars Technica (AI tag)

ไม่ต้องดึงทุกแหล่งให้ครบทุกวัน — ถ้าแหล่งไหนไม่มีข่าวใหม่ก็ข้ามได้ เป้าหมายคือได้ข่าวสำคัญจริงๆ ประมาณ 4-10 ข่าวต่อวัน ไม่ใช่ทุกข่าวที่เจอ ให้ความสำคัญกับ: เปิดตัวโมเดล/ฟีเจอร์ใหม่, ดีลใหญ่/ระดมทุน, งานวิจัยที่มีผลกระทบจริง, กฎหมาย/นโยบายสำคัญ — ข้ามข่าวเล็กน้อย/ข่าวลือ/บทความความเห็นทั่วไป

**ผู้ใช้ขอเน้นเป็นพิเศษ: ข่าวที่ "น่าอ่าน" และข่าวประเภทฟีเจอร์/เครื่องมือใหม่ที่ใช้งานได้จริง**
- เวลาเจอข่าวเปิดตัวฟีเจอร์ใหม่/โมเดลใหม่ ให้จับประเด็นว่า **"ใช้ทำอะไรได้บ้าง"** เป็นหลัก ไม่ใช่แค่บอกว่า "บริษัท X เปิดตัวฟีเจอร์ Y" เฉยๆ เช่น ถ้ามีฟีเจอร์ใหม่ที่ทำเพลงได้ ทำภาพได้ แก้วิดีโอได้ ตอบคำถามจากรูปได้ ฯลฯ ต้องระบุความสามารถจริงพวกนี้ให้ชัดในสรุป ไม่ใช่พูดลอยๆ
- ให้ความสำคัญกับข่าวกลุ่มนี้มากกว่าข่าวธุรกิจ/นโยบายที่แห้งๆ อ่านไม่สนุก — ถ้าวันไหนมีข่าวฟีเจอร์ใหม่ที่คนทั่วไปเอาไปใช้ได้จริงในชีวิตประจำวัน ให้เลือกมาก่อน แม้จะไม่ได้มาจากค่ายใหญ่ที่สุดก็ตาม

## 2. กันข่าวซ้ำ
เทียบ `url` ของแต่ละข่าวที่เจอกับ `url` ใน `data/latest.json` — ถ้าซ้ำ ข้ามไป ไม่เขียนซ้ำ

## 3. สรุปเป็นไทยแต่ละข่าว
สำหรับข่าวใหม่แต่ละชิ้น เขียนเป็น object ตาม schema นี้:

```json
{
  "source": "ชื่อบริษัท/สื่อ เช่น OpenAI, TechCrunch",
  "title_th": "หัวข้อข่าวแปล/สรุปเป็นไทย กระชับ ใจความชัด",
  "summary_th": "สรุป 2-4 ประโยคภาษาไทย อ่านลื่น ไม่ใช่แปลคำต่อคำ เน้นว่า 'เกิดอะไรขึ้น' และ 'สำคัญยังไง'",
  "category": "model_release | product | funding | research | policy | other",
  "importance": "major | normal",
  "url": "ลิงก์ข่าวต้นฉบับ",
  "context_th": "(ไม่บังคับ) ดูหัวข้อ 'เชื่อมโยงเรื่องราว' ด้านล่าง"
}
```

### เชื่อมโยงเรื่องราวข้ามเวลา (`context_th`, ไม่บังคับ — ใส่เฉพาะเมื่อมีหลักฐานจริง)

ก่อนเขียนสรุปข่าวแต่ละชิ้น ให้ดู `data/latest.json` (และถ้าจำเป็นดูไฟล์เก่าใน `data/news/` เพิ่ม) ว่ามีข่าวเก่าที่ **เกี่ยวข้องกันจริงๆ** ไหม เช่น:
- เป็นข่าวต่อเนื่อง/ตอบโต้ข่าวเก่า (บริษัท A ลดราคาหลังบริษัท B ออกโมเดลถูกกว่าเมื่อ 2 สัปดาห์ก่อน)
- เป็นภาคต่อของเรื่องเดิม (บริษัทเดิมประกาศอัปเดตของสิ่งที่เคยประกาศไปก่อนหน้า)

ถ้าเจอความเชื่อมโยงจริงแบบนี้ ให้เติม `context_th` เป็นประโยคเดียวสั้นๆ อธิบายว่าเชื่อมกับข่าวไหนตอนไหน เช่น `"สืบเนื่องจากที่ Google เปิดตัวโมเดลราคาถูกกว่าเมื่อสัปดาห์ก่อน (ข่าว 4 ก.ค.)"`

**กฎสำคัญ: ห้ามเดา ห้ามมั่ว** — ถ้าไม่มีข่าวเก่าที่เชื่อมโยงกันจริงๆ ให้ **เว้นฟิลด์นี้ไปเลย** (ไม่ต้องใส่ หรือใส่เป็น string ว่าง) ดีกว่าใส่ความเชื่อมโยงที่เดาขึ้นมาเอง ผู้ใช้ระบุไว้ชัดเจนว่าไม่ต้องการการคาดเดา/ทำนาย ต้องเป็นการอ้างอิงข้อเท็จจริงที่มีอยู่แล้วเท่านั้น

### เกณฑ์ตัดสิน `importance` (สำคัญมาก — ผู้ใช้อยากให้เน้นข่าวที่สำคัญจริงๆ ให้เด่นชัด อย่าใส่ major พร่ำเพรื่อ)

ให้ `"major"` เฉพาะข่าวที่เข้าเกณฑ์ข้อใดข้อหนึ่งต่อไปนี้เท่านั้น (ปกติควรมีแค่ 0-2 ข่าว/วัน ไม่ใช่ทุกข่าว):
- โมเดล/ผลิตภัณฑ์ AI เรือธงตัวใหม่จากค่ายใหญ่ (OpenAI, Anthropic, Google/DeepMind, Meta, Microsoft, xAI) ที่เปลี่ยนสถานะการแข่งขันจริง ไม่ใช่แค่อัปเดตเล็กน้อย
- งานวิจัยที่เป็น breakthrough จริง ๆ ที่สื่อหลักหลายเจ้าพูดถึงพร้อมกัน ไม่ใช่ paper ทั่วไป
- ดีล/ระดมทุนขนาดใหญ่มาก (เช่น ระดับพันล้านดอลลาร์ขึ้นไป) หรือการควบรวมกิจการครั้งสำคัญ
- กฎหมาย/นโยบายที่กระทบวงกว้างจริง (เช่น กฎหมายผ่านสภา มีผลบังคับใช้จริง ไม่ใช่แค่ร่างหรือแนวปฏิบัติย่อย)
- เหตุการณ์ด้านความปลอดภัย/จริยธรรม AI ที่เป็นข่าวใหญ่ระดับโลก (การรั่วไหลของข้อมูล, อุบัติเหตุร้ายแรง, ฟ้องร้องสำคัญ)

ข่าวอื่นๆ ที่เหลือ (ฟีเจอร์อัปเดตทั่วไป, เวอร์ชันย่อย, ทุนระดับกลาง-เล็ก, งานวิจัยทั่วไป, บทวิเคราะห์) ให้เป็น `"normal"` — เว็บจะเอาข่าว major ไปแยกโชว์เด่นด้านบนสุดของหน้า พร้อมสัญลักษณ์ 🔥 ให้อัตโนมัติ

## 4. เขียนไฟล์ข่าววันนี้
เขียน `data/news/{today}.json` เป็น array ของ object ข้างต้น (ถ้าไฟล์นี้มีอยู่แล้วจากรอบก่อนหน้าของวันเดียวกัน ให้ merge ข่าวใหม่เข้าไปแทนที่จะเขียนทับ)

### เล่าเป็นเรื่องราว (`data/narrative/{today}.json`, ไม่บังคับ)
ถ้าวันนี้มีข่าว ≥2 ข่าว ให้เขียนไฟล์เพิ่มเติม `data/narrative/{today}.json`:

```json
{ "narrative_th": "ย่อหน้าเดียว ร้อยเรียงข่าววันนี้ทั้งหมดเป็นเรื่องเล่าที่ลื่นไหล เหมือนคอลัมนิสต์เขียนสรุปวงการ ไม่ใช่แค่เอาสรุปแต่ละข่าวมาต่อกัน" }
```

เขียนแบบมีเสียง/มุมมอง (เช่น เชื่อมโยงว่าเหตุการณ์ไหนเกิดก่อนเกิดหลัง ใครตอบโต้ใคร) แต่ **ต้องอิงข้อเท็จจริงจากข่าวที่เก็บมาเท่านั้น ห้ามแต่งเติมหรือคาดเดา** ถ้าวันนั้นมีข่าวแค่ 1 ข่าวหรือข่าวไม่เกี่ยวข้องกันเลยจนร้อยเรียงเป็นเรื่องเดียวไม่ได้ ให้ข้ามไฟล์นี้ไปเลย ไม่ต้องฝืนเขียน

## 5. รวบยอดรายสัปดาห์ (เฉพาะวันอาทิตย์)
ถ้า `{today}` เป็นวันอาทิตย์:
- รวมข่าวทั้งหมดใน `data/news/` ของสัปดาห์นี้ (จันทร์-อาทิตย์)
- เขียนสรุป `highlights` 3-5 ข้อ (ภาพรวมสัปดาห์ ไม่ใช่แค่ list หัวข้อ) เป็นภาษาไทย
- **วิเคราะห์ทิศทางแต่ละบริษัท (`directions`)**: สำหรับบริษัทที่มีข่าว ≥2 ข่าวในสัปดาห์นี้ (นับรวมสัปดาห์ก่อนๆ ถ้าช่วยให้เห็นภาพชัดขึ้นได้) ให้สรุปสั้นๆ ว่าข่าวของบริษัทนั้นเน้นไปทางไหน **โดยอ้างอิงจากสัดส่วนหมวดข่าวจริงที่นับได้เท่านั้น** (เช่น "ออกข่าวด้านลดราคา/ต้นทุน 3 จาก 4 ข่าวล่าสุด") — **ห้ามทำนายอนาคตหรือเดาเหตุผลเชิงกลยุทธ์ที่ไม่มีหลักฐาน** ถ้าข่าวของบริษัทนั้นดูกระจัดกระจายไม่มีแนวโน้มชัด ให้ข้ามบริษัทนั้นไปเลย ไม่ต้องฝืนหาแนวโน้ม
- เขียนไฟล์ `data/weekly/{ISO-week}.json` ตาม schema:

```json
{
  "week": "2026-W28",
  "start_date": "2026-07-06",
  "end_date": "2026-07-12",
  "highlights": ["...", "...", "..."],
  "directions": {
    "OpenAI": "ออกข่าวด้านโมเดลราคาประหยัดบ่อยสุด (3 จาก 4 ข่าว) เน้นแข่งขันด้านต้นทุน",
    "Anthropic": "ออกข่าวด้านธรรมาภิบาล/ความปลอดภัยเป็นหลักสัปดาห์นี้"
  },
  "items": [ /* รวมข่าวทั้งสัปดาห์ ตาม schema ข้อ 3 แต่ละชิ้นมี "date" เพิ่มด้วย */ ]
}
```

(`directions` ไม่บังคับ — ใส่เฉพาะบริษัทที่มีหลักฐานชัดจริงๆ ข้ามได้ทั้ง object ถ้าสัปดาห์นั้นไม่มีบริษัทไหนมีแนวโน้มชัดเจนพอ)

## 9. บันทึกผลการรันทุกครั้ง (บังคับเสมอ ห้ามข้าม แม้ไม่มีข่าวใหม่)
**นี่คือขั้นตอนที่สำคัญที่สุดในการทำให้ผู้ใช้มั่นใจว่า routine ทำงานจริงทุกวัน** — ก่อนจบรอบ ให้เปิด/สร้างไฟล์ `data/engine-log.md` แล้วเติมบรรทัดใหม่ต่อท้าย (ห้ามลบของเดิม) ตามรูปแบบ:

```
- {today} {เวลา UTC ตอนรัน}: พบข่าวใหม่ {n} ข่าว, เขียน data/news/{today}.json แล้ว, generate.py สำเร็จ, push แล้ว (commit {short-sha})
```

หรือถ้าค้นแล้วไม่เจอข่าวใหม่จริงๆ (หลังค้นครบทุกแหล่งใน 1. แล้วจริงๆ ไม่ใช่แค่ลองแหล่งเดียวแล้วเลิก):
```
- {today} {เวลา UTC ตอนรัน}: ไม่พบข่าวใหม่หลังค้น {รายชื่อแหล่งที่ลองจริง} — ข้ามการเขียนข่าววันนี้
```

หรือถ้าขั้นตอนไหนล้มเหลว (เช่น git push พัง, generate.py error):
```
- {today} {เวลา UTC ตอนรัน}: ล้มเหลวที่ขั้นตอน {ชื่อขั้นตอน} — {ข้อความ error จริง}
```

**จากนั้น commit + push ไฟล์ `data/engine-log.md` นี้เสมอ ไม่ว่าผลลัพธ์วันนั้นจะเป็นแบบไหนก็ตาม** (ถ้ามีข่าวใหม่ด้วย ให้รวมอยู่ใน commit เดียวกับขั้นตอน 8 ได้เลย ไม่ต้อง commit แยก) — นี่คือสิ่งเดียวที่ทำให้เห็นจาก git history ว่า routine รันจริงทุกวันหรือไม่ ถ้าไม่มี log แปลว่า routine พังตั้งแต่ต้นๆ ก่อนถึงขั้นตอนนี้

**อย่าใจร้อนสรุปว่า "ไม่มีข่าวใหม่"** — วงการ AI มีข่าวแทบทุกวันจากอย่างน้อยหนึ่งในบริษัท/สื่อที่ลิสต์ไว้ใน 1. ถ้าค้นแค่ 1-2 แหล่งแล้วไม่เจอ ให้ลองแหล่งอื่นต่อก่อนสรุปว่าไม่มี การสรุปว่า "ไม่มีข่าว" ควรเป็นเหตุการณ์ที่เกิดยากมาก ไม่ใช่ค่าเริ่มต้น

## หมายเหตุ
- ห้ามสมัครบัญชี/บริการใหม่ใดๆ ระหว่างรัน routine นี้ — ใช้เฉพาะเครื่องมือ WebSearch/WebFetch/Bash/git ที่มีอยู่แล้วเท่านั้น
- ถ้า `git push` ล้มเหลวไม่ว่าด้วยเหตุผลใด **ห้ามเงียบแล้วจบรอบไปเฉยๆ** ต้องบันทึกข้อความ error จริงลงขั้นตอน 9 (แม้จะ commit ในเครื่อง sandbox ไม่ได้เพราะ push พัง ก็ให้ใส่ข้อความ error ไว้ในสรุปข้อความตอบท้ายสุดของรอบให้ชัดเจนที่สุดเท่าที่ทำได้)
